# Полный MLOps Pipeline (работу выполнил Нестеров Дмитрий РИМ-240901)

## Цель проекта
Запуск полного MLOps pipeline в docker.

**Ключевые компоненты:**
- Мониторинг дрейфа данных — Airflow
- Автоматическое обучение моделей — PyCaret
- Регистрация моделей — MLflow
- Роутинг — Flask
- Данные — adult.csv (задача классификации заработных плат)

------------------------------------------------------------------------

## Пример использования

Собираем докер контейнер:
```bash
  docker compose up --build
```
В результате должен получиться докер контейнер, содержащий 3 докер образа, а именно mlflow, flask, airflow. 

------------------------------------------------------------------------


Для работы с конвеером необходимо использовать GET и POST запросы.
Обучения модели на тестовом наборе данных и дообучения на новых данных:
```bash
    curl -X POST "127.0.0.1:8000/api/train"
```
Получение настроек распределения предсказаний для ```STAGING``` и ```PRODUCTION``` моделей:
```bash
    curl "127.0.0.1:8000/api/ab/config"
```
Настройка распределения предсказаний для ```STAGING``` и ```PRODUCTION``` моделей:
```bash
    curl -X POST "127.0.0.1:8000/api/ab/config" -H 'accept: application/json' -H 'Content-Type: application/json' -d {'staging' : 0.3, 'production' : 0.7}
```
Получение результатов A/B тестирования для принятия решения продвижения модели в production:
```bash
    curl "127.0.0.1:8000/api/ab/report"
```
Предсказание данных моделью:
```bash
    curl -X POST "127.0.0.1:8000/api/predict" -H 'accept: application/json' -H 'Content-Type: application/json' -d {"age": 26, "workclass": "Private", "fnlwgt": 95691, "education": "HS-grad", "education-num": 9, "marital-status": "Never-married", "occupation": "Other-service", "relationship": "Unmarried", "race": "White", "sex": "Female", "capital-gain": 0, "capital-loss": 0, "hours-per-week": 22, "native-country": "Columbia"}
```

------------------------------------------------------------------------

Ежденевно DAG опрашивает сервер об дрифте данных (PSI > 0.1), если он есть запускает переобучение модели. Сам график можно посмотреть в папке images
Каждый запрос на предсказания логируется к модели. В дальнейшем эти данные используются для проверки признаков на дрифт.
По пути ```scripts/predict_data.py``` лежит простая программа, которая симулирует запросы пользователей к модели.
В папке ```images``` лежат скриншоты работы приложения.
